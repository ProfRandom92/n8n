## Usage Instructions

1. **Copy each URL** individually
2. **Paste into Make.com** RSS modules
3. **Test URLs** in browser first
4. **Adjust parameters** if needed:
   - `radius=50` (distance in km)
   - `sort=date` (newest first)
   - `l=remote` (location preference)

## Parameter Reference

### Location Options:
- `l=remote` - Remote jobs only
- `l=deutschland` - All of Germany
- `l=heidelberg` - Specific city
- `l=neckargemünd` - Your location

### Radius Options:
- `radius=25` - 25km radius
- `radius=50` - 50km radius
- `radius=100` - 100km radius

### Sort Options:
- `sort=date` - Newest first
- `sort=relevance` - Most relevant first

### Job Type Options:
- `f_JT=2` - Full-time only
- `f_JT=3` - Part-time only
- `f_JT=2,3` - Both full-time and part-time

## Troubleshooting

### Common Issues:
- **404 Error:** URL may be outdated
- **Empty Feed:** No jobs matching criteria
- **Wrong Jobs:** Adjust search parameters
- **Rate Limiting:** Add delays between requests

### Solutions:
- **Update URLs** regularly
- **Broaden search** terms if needed
- **Add delays** in Make.com workflow
- **Use multiple sources** for redundancy
